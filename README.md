# Shlink URL Shortener Production Infrastructure

This repository contains a complete, production-grade, containerized infrastructure for self-hosting a [Shlink](https://shlink.io) URL shortener on a small VPS (like AWS Lightsail, EC2, DigitalOcean, or Linode) running Ubuntu 24.04 LTS.

It is designed for reliability, security, high performance, automatic backups, and low resource overhead, capable of handling 10,000+ clicks per day and bursts of 200+ concurrent visitors.

## 3. Quick Start Guide (For Fresh VPS)

Follow these steps to deploy Shlink from scratch on a new server.

### Step 1: Update Server & Configure Firewall
Log into your Ubuntu server via SSH and run:
```bash
sudo apt update && sudo apt upgrade -y
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 443/udp
sudo ufw enable
```

### Step 2: Clone the Repository & Make Scripts Executable
```bash
git clone https://github.com/your-username/shlink-production.git /opt/shlink-production
cd /opt/shlink-production
chmod +x scripts/*.sh
```

### Step 3: Install Docker
Run the automated Docker engine installation script:
```bash
sudo ./scripts/install-docker.sh
```
*After completion, run `newgrp docker` or log out/in to apply group permissions.*

### Step 4: Configure Environment
Copy the configuration template and edit the values:
```bash
cp .env.example .env
nano .env
```
Ensure you change:
- `DB_PASSWORD` (use `openssl rand -hex 24` to generate one)
- `SHLINK_SECRET_KEY` (use `openssl rand -base64 48` to generate one)
- `SHLINK_DOMAIN` (your short URL domain, e.g. `arogy.am`)
- `LETSENCRYPT_EMAIL` (your email address)

### Step 5: Configure DNS
Create an **A Record** pointing your short domain (`arogy.am`) to your VPS public IP address in your DNS control panel (e.g. GoDaddy).
- *See [DNS Setup & Domain Configuration](docs/dns.md) for more details.*

### Step 6: Deploy Stack
```bash
./scripts/deploy.sh
```
The script will download the containers, configure database tables, verify service health, and output your **first Shlink Admin API Key**.

## 4. Post-Deployment Verification Checklist

- [ ] **Verify HTTPs**: Navigate to `https://<your-short-domain>/rest/health`. It should return `{"status":"pass", ...}` or `{"status":"ok"}`.
- [ ] **Secure API Key**: Ensure you saved the API key generated by the deploy script. If you missed it, generate a new one:
  ```bash
  docker exec -it shlink-engine shlink api-key:generate
  ```
- [ ] **Connect to Web UI**: Open the official [Shlink Web Client](https://app.shlink.io/) in your browser. Click **Add server**, enter your domain name (`https://arogy.am`), and paste your API key.
- [ ] **Generate First Short URL**: Use the Web UI (or CLI: `docker exec -it shlink-engine shlink short-url:generate https://google.com`) to shorten a link.
- [ ] **Verify Analytics**: Click the shortened link, wait 2 seconds, and inspect the Web UI to confirm that the click count, user agent, and referrer were recorded.

## 5. Operations & Maintenance Checklist

### Daily
- [ ] **Check Health**: Execute `./scripts/health-check.sh` to monitor resources and API latency.
- [ ] **Cron Backups**: Ensure the daily cron backup ran successfully (log file: `/var/log/shlink-backup.log`).

### Weekly
- [ ] **Verify Backups**: Download a `.dump` file from `./postgres/backups/` and check its size.
- [ ] **Inspect Logs**: Check Caddy access logs and engine logs for errors using `./scripts/logs.sh --lines 100`.

### Monthly
- [ ] **Update Containers**: Run `./scripts/update.sh` to safely pull newer images and perform an auto-restoration rollback if health checks fail.
- [ ] **Test Recovery**: Periodically execute the restoration script on a test machine to verify backup integrity.
